So, I checked how it looks yesterday and the good news is the Goldfish scraping logic still works, so theoretically you should be able to just clone the repo, set up the Python environment for it and then just import the function in your IDE and run it. But...now the bad ones :)

First, irritably, there's a small bug just within the function itself (that shows I didn't really use it): https://github.com/mazz3rr/mtg/blob/04e390e65ea1c4654a2b4c33527838965432daf2/mtg/deck/scrapers/goldfish.py#L326 at this line you have to remove the non-existent "supress errors" parameters. That's some residue of earlier versions of the code. You can keep throttled=True​ if you want - like me - to err on the safe side of being blocked by Goldfish servers. But, I guess the additional throttling between requests is not really needed as Selenium with each request has to not only spawn a browser, but also close a consent popup within it - which already introduces natural delays between requests. You probably could also make a simple improvement to the function of introducing some input parameter to limit the number of decks scraped from the page. Right now it just scrapes all 60 available decks. Which can take some time :)
Second, when I tried to switch to master​ branch myself and run the function, the code complained about secrets.json​ missing which I guess you'll also see. You need this file in your CWD (current working directory) even if Goldfish logic doesn't really need it. As the logic doesn't really need it it can be empty (or, just put {}​ .there to make it a valid JSON).
I wasn't able to test if there are some further issues, because at this point the code broke for me as I've already upgraded to a new scrython​ version in my environment. If you set up your Python env according to requirements.txt​ on the master​ branch you should be fine (at least in this department) though.
Now, assumming you have successfully run the function, you'll have a list of Deck​ objects. You can then export them to various formats using https://github.com/mazz3rr/mtg/blob/master/mtg/deck/export.py
You don't need this private Google Sheet that I was talking about earlier for all of this. The sheet is needed for scraping YT channels - which I mainly did. You've just interested in scraping MTGGoldfish meta page, so you don't need it.

Sorry for the code being this stubborn and unhelpful. I guess that's how hobby projects usually look :).

Take care and good luck :)
Piotr

So, getting back to the matter at hand, I see following problems:

scope: my focus was scraping YT channels that post MTG decklists in their video descriptions. Initially, I was just curious about statistical tendencies of what MTG YT channels talk about in general or what a prevailing platform to post/edit a MTG deck is or what format is most popular. Plus, statistics about those YT channels, who's a rising star, who's the most prolific poster, etc. This kind of stuff.
character: I've never put any work in making it more user-friendly for others. That's why there are no instructions on how to use it in the README. That's why anybody who'd try to replicate my setup would face some major hurdles:
* I use a private Google Sheet as a convenient frontend (to see channel stats in a nicely formatted table) and as a partial backend (as a source of channel IDs to be scraped). Code expects communication with this sheet so anybody who wants to run the scripts has to replicate this setup by himself (without clear instructions how :)). Also, I've never shared the channel IDs I've gathered throughout the work on this project, and I'm not very inclined to do so as most of those 2512 channels had been found MANUALLY 👀

current state: I had a half a year break of working on the project as I was pretty fed up (and still are) with the slopificated direction the game is going.
